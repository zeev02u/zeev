# אתר לימוד לדוגמה

זהו מאגר תוכן קטן לאתר עברי שנבנה עם Jekyll, GitHub Pages ו-Just the Docs. המטרה היא לאפשר למורה להוסיף עמודים ושיעורים ב-Markdown, כאשר GitHub Actions בונה ומפרסם את האתר.

## הוספת שיעור חדש

צרו קובץ חדש בתיקייה `lessons/`, למשל `lessons/fractions.md`, והתחילו אותו כך:

```md
---
layout: default
title: שברים
parent: שיעורים
nav_order: 3
permalink: /lessons/fractions/
---
```

לאחר ה-front matter כתבו את תוכן השיעור ב-Markdown.

## ניווט באתר

הניווט נשלט דרך שדות ה-front matter. השתמשו ב-`nav_order` כדי לקבוע סדר בתפריט, וב-`parent: שיעורים` כדי לשייך עמוד לאזור השיעורים. כתובות קבועות מוגדרות באמצעות `permalink`.

## פרסום דרך GitHub Actions

בכל דחיפה לענף `main`, התהליך `.github/workflows/pages.yml` בונה את האתר ומפרסם אותו ל-GitHub Pages. אם הפרסום נכשל, פתחו את לשונית **Actions** במאגר, בחרו את הריצה האחרונה, וקראו את השלב שסומן כנכשל.

## שינוי גרסת התבנית

התבנית מוגדרת בקובץ `_config.yml` כך:

```yaml
remote_theme: just-the-docs/just-the-docs@v0.12.0
```

שנו את הגרסה רק במכוון, לאחר בדיקת הערות השחרור של Just the Docs. אין להעתיק את קבצי המקור של התבנית לתוך המאגר אלא אם יש צורך נקודתי וברור.

## רישוי ותוכן

המאגר ציבורי, אך ציבוריות אינה מגדירה אוטומטית רישיון שימוש בתוכן המקורי של המורה. אם נדרש רישיון, יש להוסיף קובץ `LICENSE` מתאים במפורש.

## תצוגה מקומית

תצוגה מקומית עם Ruby, Bundler או Jekyll היא אפשרות מתקדמת בלבד למי שכבר מחזיק סביבה עובדת. היא אינה חלק מתהליך העבודה הנדרש בסדנה זו. הדרך המומלצת כאן היא עריכה ב-Windows, דחיפה ל-GitHub ופרסום דרך GitHub Actions.
